02·WHY ARCHITECTURE MATTERS NOW

EVERY ENTERPRISE
IS BECOMING
A RETRIEVAL ENGINE.

The shift from ten-blue-link search to agentic, retrieval-augmented applications is not a UI change. It is an infrastructure change. Every product team is now standing up the same primitive: find the right context, in the right form, fast enough for a model to use it. Today they assemble that primitive from four vendors and a Python orchestrator. The result is a retrieval stack that is brittle, expensive, and impossible to certify.

FRAGMENTATION TAX FOUR STORES.
ONE QUERY. A typical agentic application reads from full-text, vector, columnar, and session-memory stores in a single turn — over four wire protocols, four vendors, and four bills.
DATA GRAVITY CONTEXT IS
DATA-LOCAL. Logs, embeddings, metrics, and sessions describe the same events. Joining them in a Python orchestrator is an architecture mistake; the join belongs in storage.
SOVEREIGNTY YOUR DATA.
YOUR PERIMETER. Enterprise data — security telemetry, customer records, regulated content — does not leave the customer's control. The retrieval engine has to be self-hostable, top to bottom.
DETERMINISM SLOS THAT
HOLD. Agent loops amplify tail latency by the number of retrieval calls per turn. A 99th-percentile spike upstream is a user-visible stall downstream. Latency must be predictable, not average-good.

WHAT XERJ ARGUES

The right architectural answer is a unified data plane — one engine that natively understands every retrieval modality, speaks the wire protocol enterprises already use, and runs inside the customer's security boundary.

XERJ.AI 04 · ARCHITECTURAL PRINCIPLES
04·ARCHITECTURAL PRINCIPLES

SIX RULES.
ONE ENGINE.

The shape of the engine is a direct consequence of six commitments. Every internal trade-off — sharded memtables, mmap-resident segments, embedded consensus, in-process embedding proxy — is the cheapest implementation of one of these principles.

01
UNIFIED RETRIEVAL
Full-text, dense-vector, columnar logs, and agent memory share a single segment cache, a single write-ahead log, and a single permission model. There is no cross-store orchestrator because there is no cross-store boundary.
02
DROP-IN COMPATIBILITY
The primary API is the Elasticsearch 8.13 wire protocol. Existing clients, dashboards, and ingest collectors connect unchanged. Migration is a configuration change, not a rewrite.
03
DETERMINISTIC RUNTIME
Rust, no garbage collector, no JVM warm-up. Cold start is single-digit milliseconds. Tail latency is governed by code paths, not heap pressure. SLOs measured at p99.9 mean what they say.
04
SOVEREIGN BY DEFAULT
A single static binary. Air-gap installable. No mandatory cloud dependency. Identity, encryption keys, and audit data live wherever the customer's policy says they live.
05
EXPLAINABLE BY CONSTRUCTION
Every query produces an explain-plan with cost and row counts. Every privileged action produces an audit entry. Every release ships with reproduction scripts for every published number.
06
ONE OPERATIONAL SURFACE
One config file. One metrics endpoint. One audit trail. One backup story. The platform that runs SIEM also runs RAG, observability, and agent memory — same on-call rotation, same dashboards, same upgrade procedure.

XERJ.AI 08 · STORAGE & DURABILITY
08·STORAGE & DURABILITY

SHARDED MEMTABLE.
SEALED SEGMENTS.

Writes hit a sharded in-memory table to spread contention across cores. Durability is anchored in a single write-ahead log with monotonic sequence numbers, fsynced once per batch. Periodic flushes drain the memtable into immutable segments — compressed, memory-mapped, and merged in the background. Restart replays the log; orphan segments are re-attached on open.

WRITE PATH DURABLE STATE BACKGROUND TASK
DURABILITY
Every accepted batch is persisted to the WAL with fsync and a per-batch CRC before the client sees a 2xx. Crash recovery replays from the last checkpoint, idempotent per sequence number.
BACK-PRESSURE
When the WAL queue exceeds its watermark, ingest returns 429 with a structured retry hint. Producers slow down; the engine never silently drops.
RECOVERY
Open re-scans the data directory and re-attaches any orphan segments left by a concurrent flush and exit. Restart on a five-million-document workload preserves every accepted document.

XERJ.AI 09 · VECTOR SUBSYSTEM
09·VECTOR SUBSYSTEM

HNSW.
QUANTIZED.
RE-RANKED.

The vector engine is an in-process HNSW graph with three distance metrics, three quantization levels, and an optional full-precision re-ranking pass. Recall and latency are runtime-tunable through a single parameter, so the same index can serve a low-latency conversational lookup and a high-recall offline batch from the same on-disk artifact.

GRAPH TRAVERSAL CANDIDATE SET OPTIONAL RE-RANK
QUANTIZATION
Three modes: float32 baseline, scalar 8-bit (4× compression), nibble (8× compression). The HNSW traversal uses the quantized form; an optional float32 re-ranking pass restores precision on the final top-K.
DISTANCE METRICS
Cosine, L2, and dot product. Per-index choice. The metric is part of the on-disk format header; a loader rejects any index with a mismatched metric or format version.
RECALL · LATENCY DIAL
A single runtime parameter trades recall for latency without rebuilding the index. Low-latency UIs and high-recall analytics share one artifact.

XERJ.AI 10 · CLUSTER & STATE
10·CLUSTER & STATE

EMBEDDED CONSENSUS.
NO CO-PROCESS.

Cluster metadata — index schemas, shard assignments, the node roster — is replicated by an embedded Raft state machine. There is no separate ZooKeeper, etcd, or Consul to deploy, license, page on, or upgrade in lock-step with the engine. The data plane stays out of the consensus path: only metadata changes take a Raft round-trip.

DATA TRAFFIC RAFT (METADATA) HEARTBEAT · GOSSIP
METADATA SCOPE
Indices, schemas, shard placement, node membership. Replicated by Raft. Reads of metadata never block on consensus; the leader acknowledges, and followers serve.
DATA SCOPE
Documents, segments, vectors, and logs flow over the data ports. They are not Raft-replicated — replication for durability is per-index, governed by the metadata that Raft owns.
FAILURE MODEL
A leader loss costs one election round-trip on the metadata plane only. Reads and writes against existing indices continue uninterrupted.

XERJ.AI 11 · SECURITY & COMPLIANCE
11·SECURITY & COMPLIANCE

TAMPER-EVIDENT.
RBAC-NATIVE.

Every privileged operation is a journal entry in an append-only audit log whose entries chain by SHA-256. Tampering with any entry breaks the chain at that point and at every subsequent entry. A single endpoint verifies the chain end-to-end; operators pin known-good chain heads externally for cross-restart attestation.
