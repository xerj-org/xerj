## Summary Themes

**Performance**: Results are workload-dependent and benchmark-source-dependent. For aggregations and date histograms, OpenSearch is dramatically faster in independent testing; for text search, Elasticsearch leads. Vendor benchmarks are conflicted. Multiple version-specific regressions (2.13, 2.16, 2.17, 3.1) show instability across upgrades.

**Cost**: AWS managed OpenSearch costs are frequently higher than expected. Serverless pricing ($345/month minimum with zero load) surprises users expecting Lambda-style per-query billing. Searchable snapshots and free security features are genuine cost advantages vs Elastic Cloud.

**Clustering**: Large clusters (30+ nodes, 5,000+ indices) show consistent instability — transport timeouts, unresolved shard reallocation, memory pressure, split-brain risks. Cluster state management at scale is a recurring pain point.

**Feature Gap**: Forked at Elasticsearch 7.10 (April 2021), OpenSearch took 4 years to reach a major version release (3.0, May 2025). Missing features include TSDS, enrich processors, mature RAG/ESRE tooling, autocomplete, and observability primitives. Plugin ecosystem is less mature.

**Migration**: Viable for Elasticsearch 6.8–7.10 users. Plugin compatibility, config format differences, and data reindexing time (12+ hours for 100GB) are practical obstacles. API drift past 7.10 is an ongoing concern.

**Serverless**: Genuinely serverless in name only — OCU-based billing starts at $345/month minimum. Cold-start scaling latency when collections resume is unresolved. Backup/restore and higher capacity limits arrived late (2024–2025).

**Trajectory**: Accelerating rapidly. OpenSearch 3.0–3.3 delivered 11x performance improvement vs 1.3. Linux Foundation governance (Sept 2024) and 1 billion downloads signal legitimate ecosystem momentum. AWS managed version lag of ~6 months behind open-source releases.
