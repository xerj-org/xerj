## Key Themes by Product

### Quickwit
- **Strengths**: 90% cost reduction vs Elasticsearch for log/trace workloads; true S3-native stateless architecture; Rust efficiency; OpenTelemetry native; ES API compatible
- **Weaknesses**: Cold query latency (S3 round-trips); no ML/fuzzy/scoring; append-only only; PostgreSQL metastore as SPOF risk; low QPS ceiling; community much smaller than ES
- **Risk**: Datadog acquisition (Jan 2025) — future open-source commitment uncertain despite Apache 2.0 pledge

### ClickHouse
- **Strengths**: 4–30x query speed vs ES; 5–11x cheaper infrastructure; full SQL; native replication; now has GA full-text search; SharedMergeTree on S3 in cloud
- **Weaknesses**: Requires batched inserts; schema decisions hard to change; ZooKeeper/Keeper complexity at scale; requires Kibana customization or HyperDX; still not pure S3-native (cloud only for SharedMergeTree)
- **Risk**: At extreme scale (30PB+) single cluster management becomes difficult; Kwai migrated away to Apache Doris

### Grafana Loki
- **Strengths**: Very low storage cost; excellent Kubernetes integration; Grafana ecosystem; metadata-only indexing simplicity; TSDB now GA
- **Weaknesses**: Full-text search is ~1000x slower than ES; ingester OOM is common production issue; high cardinality kills performance; LogQL less expressive than ES Query DSL; ruler is single-threaded; bloom filters still experimental
- **Risk**: AGPLv3 license for enterprise use; advanced features locked behind Grafana Enterprise ($25K+/year)
