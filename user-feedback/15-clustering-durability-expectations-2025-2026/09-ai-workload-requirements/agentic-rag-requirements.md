## Key Themes

1. **Multi-model convergence**: Enterprise AI workloads require vector, graph, relational, and document data in a single ACID-transactional engine. Standalone vector DBs are losing ground to integrated platforms.

2. **Latency tiering**: Human-interactive RAG targets <2s total; agentic batch workflows care more about P99 across 100 parallel queries than P50; voice agents need <200ms.

3. **Scale inflection points**: pgvector handles 5-10M vectors comfortably; HNSW requires full in-memory graph (4TiB per 1B vectors); distributed/GPU approaches needed at 100M+ vectors.

4. **Memory architecture is a hard engineering problem**: Multi-tiered (session → episodic → semantic) with extraction pipelines, decay policies, PII redaction, and user_id isolation. Most 2025 production failures traced to missing durable memory layer.

5. **Real-time freshness is critical**: 40% of RAG failures attributed to stale data; healthcare and finance require near-continuous index updates, not daily batch reindexing.

6. **Semantic caching delivers outsized ROI**: 70-90% cost reduction documented in production; requires ANN search, embedding models, and vector storage as first-class infrastructure.

7. **Hybrid search is table stakes**: 85% of enterprises report accuracy improvement; combining dense vector + BM25 keyword retrieval with reranking is now the production baseline.
