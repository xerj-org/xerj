## Key Takeaways

1. **Hybrid is the production standard**: 72% of production RAG systems use hybrid search as of early 2026, up from near zero in 2023.
2. **Neither approach alone is sufficient**: Vector search fails on exact identifiers; keyword search fails on semantic intent. Production systems require both.
3. **Score normalization is unsolved**: Min-max, Z-score, and RRF all have tradeoffs; no universally correct approach exists.
4. **Latency target is 100–300 ms**: Users abandon after 3 seconds; the overhead of dual-index retrieval + reranking must fit within this budget.
5. **Tuning is continuous**: Alpha weights, candidate set size, and normalization strategy drift with data changes and must be monitored.
6. **Operational complexity is a barrier**: Dual indexing, embedding model hosting, reranking sidecars, and observability create substantial infrastructure overhead.
7. **Database-native hybrid search is emerging**: Postgres (pgvector + BM25), Spanner, MariaDB, and SQL Server are adding hybrid search natively, reducing the need for separate vector databases.
